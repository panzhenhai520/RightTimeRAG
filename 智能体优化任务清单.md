# 智能体优化任务清单

## 1. 全局补充和硬边界

以下约束适用于每一个任务，开发时必须先检查，测试时必须复核。

| 全局项 | 要求 |
|---|---|
| 开发后立即测试 | 每完成一个任务，必须立即执行对应测试，通过后才能进入下一个任务。每日巡检和发版前回归只是补充，不替代即时测试。 |
| 不影响聊天助手 | 不改聊天助手现有回答正文格式；不改 retrieving / thinking / answer 输出样式；不改普通问答流式输出节奏。 |
| 事件隔离 | 智能体 run event 必须使用独立命名空间、独立 store、独立恢复逻辑，不能混入聊天助手会话事件。 |
| 状态隔离 | 前端不能复用同一个临时状态对象同时承载聊天助手和智能体运行状态，避免历史会话污染。 |
| 后端为安全边界 | 前端连线校验只是体验，发布校验、运行前校验和权限校验必须在后端存在。 |
| 本地优先 | 默认依赖本地 ASR/TTS/LLM/Embedding 服务，运行时不应默认依赖外网。 |
| 大对象处理 | 大文本、大表格、大音频、大文件不能写入事件表，必须走 Artifact 或摘要。 |
| 确定性计算 | Excel、SQL、聚合、评分公式、成绩计算必须由代码完成，不由 LLM 猜测。 |
| 旧流程兼容 | 旧 DSL 必须能加载；缺少新 schema 的旧节点可以 warning，不应直接崩溃。 |
| 回退开关 | 智能体长任务、事件重放、队列 worker 等改动必须保留回退或灰度开关。 |

## 2. 任务 0：基线冻结

### 全局补充

本任务不改功能，只确认当前 v2 代码作为后续优化基线。不得修改聊天助手链路，不得引入智能体事件和聊天助手事件混用。

### 目标

确认当前智能体平台 v2 能力完整可用，作为后续任务的回归基线。

### 改进内容

1. 运行现有智能体平台回归脚本。
2. 运行前端生产构建。
3. 记录当前提交号、分支、工作区状态。
4. 明确后续任务都基于本基线增量开发。

### 边界

1. 不新增节点。
2. 不改 UI。
3. 不改 SSE。
4. 不改聊天助手。
5. 不清理用户已有未提交改动。

### 达到标准

1. `newapp/ragflow` 工作区状态清楚。
2. 后端回归全部通过。
3. 前端构建通过。

### 测试方法

```bash
cd /home/xsuper/app/newapp/ragflow
tools/run_agent_platform_regression.sh
cd web && npm run build
```

## 3. 任务 1：节点 Manifest 和标准类型

### 全局补充

Manifest 是后续连线校验、节点库、发布校验、集成测试的基础。不能因为新增 manifest 破坏旧 DSL；缺字段必须自动补默认值。

### 目标

让每个节点都能向前端和后端统一声明自己的输入、输出、配置、能力、依赖和风险等级。

### 改进内容

1. 增加或补齐节点 manifest：
   - `operator`
   - `category`
   - `input_schema`
   - `output_schema`
   - `config_schema`
   - `runtime_capabilities`
   - `risk_level`
   - `requires_service`
2. 标准类型补齐：
   - `String`
   - `Number`
   - `Boolean`
   - `JSON`
   - `FileAsset`
   - `FileAsset[]`
   - `TextDocument`
   - `TextChunk[]`
   - `TableData`
   - `SQLResult`
   - `Artifact`
   - `Artifact[]`
   - `AudioAsset`
   - `VoiceReply`
   - `AgentRunRef`
   - `MeetingContext`
   - `ScoreRubric`
   - `ScoreResult`
   - `ChartSpec`
3. 后端提供统一 manifest 查询能力。
4. 前端节点库和变量选择器读取同一份 manifest。

### 边界

1. 本任务只做声明和读取，不重做画布交互。
2. 不强制所有旧节点一次性写完整配置表单。
3. 不改变节点执行结果的业务含义。

### 达到标准

1. 核心节点能导出 manifest。
2. 缺 manifest 的旧节点能自动兼容。
3. manifest 能表达本地服务依赖，例如 `cosyvoice3`、`qwen3_asr`、`xinference`。

### 测试方法

1. 增加 manifest 单元测试。
2. 验证旧 DSL 加载。
3. 运行：

```bash
tools/run_agent_platform_regression.sh
```

## 4. 任务 2：后端连线类型校验

### 全局补充

后端校验是安全边界。前端即使允许了错误连线，发布时也必须阻断。不能影响聊天助手普通问答。

### 目标

发布前检查智能体流程里所有边的类型是否兼容，避免运行后才失败。

### 改进内容

1. 在 `AgentValidationService` 中实现类型兼容矩阵。
2. 校验 source output 到 target input。
3. 错误信息包含：
   - 源节点
   - 源输出
   - 源类型
   - 目标节点
   - 目标输入
   - 目标类型
   - 修复建议
4. 旧 DSL 缺端口信息时给 warning，不直接阻断。

### 边界

1. 不改变节点执行逻辑。
2. 不在运行时隐式转换危险类型。
3. 不允许 `Artifact` 直接进 LLM prompt，必须经过读取/解析节点。
4. 不允许 `FileAsset` 直接进 `TextDocument` 输入，必须经过 `FileParser`。

### 达到标准

1. 合法连线可以发布。
2. 非法连线发布失败。
3. warning 不破坏历史流程。

### 测试方法

1. 单元测试合法连线。
2. 单元测试非法连线。
3. 单元测试旧 DSL 兼容。
4. 运行：

```bash
tools/run_agent_platform_regression.sh
```

## 8. 任务 6：表格、计算和图表节点

### 全局补充

表格读取、分数计算、权重合并、历史趋势统计必须由代码完成，不能交给 LLM 猜。

### 目标

支持 Excel/CSV 读取、列识别、聚合、公式计算、图表规格生成和导出。

### 改进内容

1. 完善或新增：
   - `ExcelRead`
   - `ColumnDetect`
   - `TableFilter`
   - `TableAggregate`
   - `NumberCalculate`
   - `ExcelExport`
   - `ChartSpecBuilder`
2. 支持权重评分公式：

```text
综合评分 = 自评得分 * 自评权重 + 外部裁判得分 * 外部权重
```

3. 生成图表规格：
   - 折线图：历史成绩趋势
   - 雷达图：发音、流利度、准确率、节奏等多维评分

### 边界

1. 第一版支持 xlsx/csv。
2. 不支持宏。
3. 不支持复杂透视表。
4. 图表节点输出 `ChartSpec`，具体渲染由前端或报告节点完成。

### 达到标准

1. 能读取学生名单 Excel。
2. 能计算综合评分。
3. 能输出图表规格。
4. 能导出 xlsx。

### 测试方法

## 9. 任务 7：数据库安全写入和动态建表节点

### 全局补充

现有 `ExeSQL` 默认只读，不能直接放开写库。动态建表和写成绩必须通过受控节点实现，不能让用户输入任意 DDL/DML。

### 目标

支持智能体在受控范围内为教学活动动态建立数据表，写入活动日志和学生成绩。

### 改进内容

1. 新增受控数据库节点：
   - `ScopedDBConnector`
   - `SafeTableEnsure`
   - `SafeRecordInsert`
   - `SafeRecordUpdate`
   - `SafeRecordQuery`
2. 限制表名命名空间，例如：

```text
agent_{agent_id}_teaching_activity
agent_{agent_id}_student_score
```

3. 动态建表只能使用预定义 schema 模板。
4. 写入字段必须来自结构化输入。
5. 查询输出为 `SQLResult` 或 `TableData`。

### 边界

1. 不改 `ExeSQL` 只读默认策略。
2. 不允许任意用户 SQL 写库。
3. 不允许跨租户建表。
4. 不允许写系统表。
5. 第一版建议只支持测试 SQLite 或租户隔离数据库 schema。

### 达到标准

1. 教学活动表可自动确保存在。
2. 学生成绩表可自动确保存在。
3. 活动日志和成绩可写入。
4. 越权表名、危险字段、非模板 DDL 被阻断。

### 测试方法

1. 动态建表成功测试。
2. 写入活动日志测试。
3. 写入成绩测试。
4. 危险表名阻断测试。
5. 跨租户访问阻断测试。
6. 运行：

```bash
tools/run_agent_platform_regression.sh
```

## 10. 任务 8：LLM、教学方案和结构化评分节点

### 全局补充

LLM 负责生成教学方案、解释和自然语言反馈，不负责最终分数计算。评分必须结构化输出并由代码校验。

### 目标

支持智能体模拟真人老师的教学思维：制定教学方案、分步骤教学、评价学生朗读、生成反馈。

### 改进内容

1. 完善或新增：
   - `PromptTemplate`
   - `LLM`
   - `StructuredLLM`
   - `ReportComposer`
   - `SummaryNode`
   - `ScoreRubricBuilder`
   - `PronunciationJudge`
2. 评分维度：
   - 发音准确率
   - 单词完整度
   - 流利度
   - 节奏
   - 重音
   - 语调
   - 跟读完成度
3. 输出结构：

```json
{
  "teacher_plan": "...",
  "teaching_steps": [],
  "self_score": 86.5,
  "rubric_scores": {
    "pronunciation": 88,
    "fluency": 82,
    "rhythm": 85
  },
  "feedback": "...",
  "next_step": "..."
}
```

### 边界

1. LLM 自评分可以作为一个输入，但不能直接成为最终成绩。
2. 结构化输出必须校验 schema。
3. 长文本报告走 Artifact。

### 达到标准

1. 能生成教学方案。
2. 能生成教学步骤。
3. 能输出结构化评分。
4. schema 不合法时能重试或失败可定位。

### 测试方法

1. fake LLM 固定输出测试。
2. schema 校验测试。
3. 评分字段缺失测试。
4. 运行：

```bash
tools/run_agent_platform_regression.sh
```

## 11. 任务 9：语音 ASR/TTS 节点

### 全局补充

语音节点默认连接本地服务。运行时不应依赖外网。服务不可用时发布校验或运行前健康检查应明确失败。

### 目标

支持智能体生成老师朗读音频，并识别学生语音输入。

   - `AudioInput`
   - `TTSGenerate`
   - `ASRTranscribe`
   - `VoiceReplyOutput`

## 15. 任务 13：最终一次集成测试 - 真人英语老师智能体

### 全局补充

这是最终主集成测试。它必须覆盖所有新增节点类别，并验证智能体能从输入、教学、语音、评分、外部评分、数据库写入、报告输出完整闭环。测试不得影响聊天助手页面和普通问答输出。

### 是否算最终集成测试

算。你描述的“模拟真人老师思维的智能体”可以作为最终集成测试，但要确保流程里显式纳入以下环节，才能覆盖所有新增节点类别：

1. 教学材料上传和解析，用于覆盖文件、OCR、检索、引用节点。
2. 学生名单或历史成绩 Excel，用于覆盖表格节点。
3. 动态建表、写活动日志、写学生成绩，用于覆盖受控数据库写入节点。
4. 老师朗读 TTS 和学生朗读 ASR，用于覆盖语音节点。
5. 外部裁判评分 Webhook 或人工补录，用于覆盖外部输入和人工确认节点。
6. 多角色并发分析，用于覆盖多智能体节点。
7. 图表和报告导出，用于覆盖输出物节点。

如果只输入一段文字和一段学生语音，不上传教学材料、不接外部评分、不写数据库、不生成图表报告，就只能算语音教学链路测试，不能算覆盖所有新增节点的最终集成测试。

### 智能体名称

`真人英语老师教学评测智能体`

### 输入

| 输入 | 类型 | 说明 |
|---|---|---|
| 教学目标文本 | `String` | 例如：教会学生念一段英文 |
| 英文朗读文本 | `String` | 例如：The quick brown fox jumps over the lazy dog. |
| 学生语音 | `AudioAsset` | 学生跟读录音 |
| 教学材料 | `FileAsset[]` | PDF/DOCX/图片，可选但集成测试必须提供 |
| 学生名单/历史成绩 | `FileAsset` | Excel fixture，集成测试必须提供 |
| 外部裁判评分 | `JSON` | Webhook 或人工补录 |
| 评分权重 | `JSON` | 例如：自评 0.6，外部评分 0.4 |

### 流程

1. `Begin/FormInput` 接收教学目标、英文文本、学生信息、权重配置。
2. `FileUpload` 接收教学材料和学生历史成绩 Excel。
3. `FileParser/DocxReader/PDFPageExtractor/OCRNode` 解析教学材料。
4. `TextChunker/HybridRetriever/CitationFormatter` 检索相关教学方法和材料片段。
5. `ExcelRead/ColumnDetect/TableFilter` 读取学生名单和历史成绩。
6. `PromptTemplate/LLM/StructuredLLM` 生成教学方案和教学步骤。
7. `TTSGenerate` 生成老师示范朗读音频。
8. `ASRTranscribe` 将学生语音识别成文字。
9. `PronunciationJudge/StructuredLLM` 对学生朗读进行自评分和维度评分。
10. `WebhookInput/ExternalScoreReceiver/HumanReview` 接收外部裁判评分或人工补录。
11. `NumberCalculate` 按权重公式合并自评分和外部评分。
12. `ScopedDBConnector/SafeTableEnsure` 动态确保教学活动表和学生成绩表存在。
13. `SafeRecordInsert` 写入教学活动日志。
14. `SafeRecordInsert` 写入学生成绩。
15. `SafeRecordQuery` 查询历史成绩。
16. `ChartSpecBuilder/ChartRenderer` 生成历史趋势图和多维评分图。
17. `ReportComposer/DocGenerator` 生成带图表的历史分析报告。
18. `ArtifactPackager/Message/VoiceReplyOutput` 输出报告、音频、综合评分和教学反馈。

### 数据库表模板

只允许受控建表，不能执行任意 SQL。

教学活动表：

```sql
agent_{agent_id}_teaching_activity(
  id,
  tenant_id,
  student_id,
  activity_id,
  target_text,
  asr_text,
  teacher_plan,
  self_score,
  external_score,
  final_score,
  created_at
)
```

学生成绩表：

```sql
agent_{agent_id}_student_score(
  id,
  tenant_id,
  student_id,
  activity_id,
  score_type,
  pronunciation,
  fluency,
  rhythm,
  intonation,
  final_score,
  created_at
)
```

### 综合评分公式

```text
综合评分 = 自评分 * self_weight + 外部裁判评分 * external_weight
```

约束：

1. `self_weight + external_weight = 1`。
2. 评分范围 `0-100`。
3. 权重和评分由代码校验。
4. 外部评分缺失时按降级策略计算，例如自评分权重临时为 1。

### 边界

1. 不使用 `ExeSQL` 执行写库。
2. 动态建表只能用受控模板。
3. 不访问外网评分服务，外部裁判使用本地 mock webhook。
4. 不把音频和报告写入事件表。
5. 不影响聊天助手。
6. 不依赖真实学生数据，使用测试 fixture。

### 达到标准

1. 能自动生成教学方案和步骤。
2. 能生成老师朗读音频。
3. 能识别学生语音为文字。
4. 能根据识别文字计算准确率和多维评分。
5. 能显示评分规则。
6. 能接收外部裁判评分。
7. 能按权重合并综合评分。
8. 能动态建立受控数据库表。
9. 能写入活动日志和学生成绩。
10. 能查询历史成绩。
11. 能生成带图表的历史分析报告。
12. 能输出报告下载、音频输出和最终反馈。
13. 失败时能通过 run trace 定位失败节点。

### 测试方法


测试脚本必须完成：
